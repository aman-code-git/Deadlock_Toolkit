Operating Systems — Deadlock Theory

Complete reference for your viva examination

01

What is Deadlock?

A deadlock is a condition where a set of processes are permanently blocked because each process holds at least one resource and is waiting for a resource held by another process in the set.

P1 holds R1 → waits for R2
P2 holds R2 → waits for R1
∴ Circular wait → DEADLOCK
02

4 Necessary Conditions (Coffman, 1971)

1Mutual Exclusion

At least one resource must be non-sharable — only one process uses it at a time.

2Hold and Wait

A process holds ≥1 resource while waiting for additional resources held by others.

3No Preemption

Resources cannot be forcibly taken from a process — only voluntarily released.

4Circular Wait

A closed chain {P0, P1, …, Pn} where P0 waits for P1, P1 for P2, …, Pn for P0.

03

Banker's Algorithm (Dijkstra, 1965)

A deadlock avoidance algorithm. Before granting any resource request, it simulates the allocation and checks if the system remains in a safe state.

1 Compute Need = Max − Allocation
2 Set Work = Available, Finish[i] = false for all i
3 Find process Pi where Finish[i]=false AND Need[i] ≤ Work
4 If found: Work += Allocation[i], Finish[i] = true → go to step 3
5 If all Finish[i]=true → SAFE STATE, else UNSAFE

05

Resource Allocation Graph (RAG)

A directed graph G = (V, E) where V = {processes P} ∪ {resources R}.

  • Assignment edge: Ri → Pj means resource Ri is allocated to Pj
  • Request edge: Pi → Rj means Pi is requesting resource Rj
  • Deadlock detection: If the graph contains a cycle AND each resource in the cycle has only one instance → deadlock
